# DevKB - 개발 문제 해결 나눗셈지

실무 개발에서 마주치는 75가지 문제를 체계적으로 해결하기 위한 완벽한 가이드입니다.

## 프로젝트 구조

```
devkb/
├── 01-patterns/          # 30개 설계 패턴 문서
├── 02-errors/            # 80개 오류 카탈로그
├── 03-flows/             # 25개 해결 플로우
├── 04-prompts/           # 45개 프롬프트 블록
│   ├── clarify/          # 문제 파악 (10개)
│   ├── reproduce/        # 문제 재현 (10개)
│   ├── diagnose/         # 원인 진단 (10개)
│   ├── patch/            # 해결책 (10개)
│   └── verify/           # 검증 (8개)
├── 05-references/        # 6개 참조 가이드
├── 06-mappings/          # 5개 맵핑 문서
└── README.md             # 이 파일
```

## 3가지 진입점

### 1. 문제별 진입 (가장 빠름)

**상황**: "에러가 발생했는데 어떻게 해야 하나?"

1. 오류 ID 또는 오류 메시지 찾기
2. `/06-mappings/error-to-flow-map.md`에서 해결 플로우 ID 확인
3. `/03-flows/F-##-*.md` 파일 열기
4. 단계별 해결책 따라하기

**예**: "Hydration failed" 오류
→ E-07 찾기 → F-03 플로우 열기 → 해결

### 2. 기술별 진입 (체계적)

**상황**: "React 성능 최적화 가이드가 필요해"

1. `/06-mappings/tech-filter-index.md`에서 기술 선택
2. 해당 기술의 플로우 목록 확인
3. 필요한 플로우 선택 후 진행

**예**: Frontend React 기술
→ F-01, F-02, F-03, F-14, F-15, F-21

### 3. 키워드 검색 (브라우징)

**상황**: "데이터베이스 성능 문제를 어디서부터 시작할까?"

1. `/06-mappings/search-keyword-index.md`에서 키워드 검색
2. 관련 플로우 확인
3. 플로우 열기

**예**: "데이터베이스 성능"
→ F-09 (N+1), F-10 (느린 쿼리), F-11 (커넥션 풀)

## 플로우 사용법

### 각 플로우의 8단계 구조

1. **증상 고정**: 문제가 정말 무엇인지 확인
2. **재현**: 문제를 반복해서 재현하기
3. **범위 축소**: 가능한 원인 종류 파악
4. **증거 수집**: 로그, 메트릭 등 실제 데이터 수집
5. **원인 후보 정렬**: 확률과 복잡도로 가설 순서 정하기
6. **수정안 선택**: 실제 구현 방법 (여러 옵션 제시)
7. **검증**: 문제가 정말 해결되었는지 확인
8. **재발 방지**: 같은 문제 반복 방지 대책

### 각 단계별 활용 프롬프트

| 단계 | 프롬프트 블록 | 용도 |
|------|-----------|------|
| 1-2 | PB-CL | 문제 명확히 하기 |
| 2-3 | PB-RP | 재현 방법 확인 |
| 3-5 | PB-DG | 원인 진단 |
| 6-7 | PB-PA | 해결책 구현 |
| 7-8 | PB-VF | 검증 및 테스트 |

## 사용 시나리오

### 시나리오 1: "갑자기 에러가 떴다"

```
1. 에러 메시지 읽기
2. error-to-flow-map.md에서 해결 플로우 찾기
3. 해당 플로우의 8단계 따라하기
4. 단계별 참조 프롬프트 활용
```

**소요 시간**: 15-30분

### 시나리오 2: "성능이 떨어졌다"

```
1. tech-filter-index.md에서 기술 선택 (DB, Frontend, etc)
2. 해당 기술의 모든 플로우 검토
3. 가장 가능성 높은 플로우부터 진단
4. 각 플로우의 "증거 수집" 단계 실행
```

**소요 시간**: 30-60분

### 시나리오 3: "미리 예방하고 싶다"

```
1. 현재 기술 스택 파악 (React, Node.js, PostgreSQL 등)
2. tech-filter-index.md에서 관련 플로우 확인
3. 각 플로우의 "재발 방지" 섹션 읽기
4. 체크리스트 항목들을 코드 리뷰에 추가
```

**소요 시간**: 2-3시간

## 효과적 사용 팁

### 1. 빠른 검색

```bash
# 플로우 파일명으로 검색
ls 03-flows/ | grep 쿼리
# → F-09-n-plus-one-fix.md, F-10-slow-query-diagnosis.md

# 키워드로 검색
grep -r "메모리 누수" .
# → F-15-memory-leak.md 찾음
```

### 2. 프롬프트 활용

각 단계에서 제시된 프롬프트 블록을 AI 도구(Claude, ChatGPT)에 입력하면:
- 더 정확한 진단 가능
- 자동 코드 생성
- 테스트 케이스 작성

### 3. 팀 온보딩

새 팀원이 처음 마주친 에러:
1. 문제 분석 (Clarify)
2. 해당 플로우 한 번 따라해보기
3. 같은 유형 문제 해결 능력 습득

### 4. 사후 분석 (Post-mortem)

프로덕션 장애 원인 분석:
1. error-to-flow-map.md로 관련 플로우 찾기
2. 플로우의 "증거 수집" 섹션 참고
3. 실제 발생한 원인 문서화
4. "재발 방지" 섹션의 대책 실행

## 피드백 및 개선

이 문서를 사용하며 발견한 사항:

- **누락된 플로우**: 특정 문제가 카버되지 않는 경우
- **개선 사항**: 설명이 불명확하거나 예제가 필요한 부분
- **새로운 기술**: 최신 기술에 대한 플로우 추가

이러한 피드백은 DevKB를 더욱 유용하게 만듭니다.

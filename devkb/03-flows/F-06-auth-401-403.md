# 인증/인가 오류 분기 해결

401(인증) 및 403(인가) 오류를 올바르게 처리하고 사용자에게 적절한 응답을 제공합니다.

## 1단계: 증상 고정

- "401 Unauthorized" 오류 발생
- "403 Forbidden" 오류 발생
- 인증되지 않은 사용자가 리소스 접근 시도
- 권한이 없는 사용자가 작업 시도
- 토큰 만료 시 적절한 처리 부재

## 2단계: 재현

```javascript
// ❌ 401 vs 403 구분 미흡
fetch('/api/protected', {
  headers: {
    'Authorization': 'Bearer invalid-token'
  }
})
  .then(res => {
    if (!res.ok) {
      console.error('Error'); // 401과 403을 구분하지 않음
    }
  });

// ❌ 토큰 없이 요청
fetch('/api/protected')
  // 401 또는 403 모두 같은 처리
```

## 3단계: 범위 축소

인증/인가 오류 유형:

1. **401 Unauthorized**: 인증 정보 없음 또는 유효하지 않음
2. **403 Forbidden**: 인증은 됐지만 권한 부족
3. **토큰 만료**: 유효했으나 시간 초과
4. **권한 부족**: 사용자는 인증됐으나 특정 리소스/작업 불가
5. **역할 기반 접근 제어(RBAC)**: 역할별 권한 관리

## 5단계: 원인 후보 정렬

| 원인 | 확률 | 복잡도 |
|------|------|--------|
| 토큰 누락 | 매우높음 | 낮음 |
| 토큰 만료 | 높음 | 중간 |
| 잘못된 토큰 | 높음 | 낮음 |
| 권한 불일치 | 높음 | 중간 |
| 역할 설정 오류 | 중간 | 중간 |

## 6단계: 수정안 선택

### 수정안 1: 서버 인증 미들웨어 (Express)

```javascript
const express = require('express');
const jwt = require('jsonwebtoken');

const app = express();

// 토큰 검증 미들웨어
const authenticateToken = (req, res, next) => {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1]; // "Bearer TOKEN"

  if (!token) {
    return res.status(401).json({ error: 'No token provided' });
  }

  jwt.verify(token, process.env.JWT_SECRET, (err, user) => {
    if (err) {
      if (err.name === 'TokenExpiredError') {
        return res.status(401).json({ error: 'Token expired' });
      }
      return res.status(401).json({ error: 'Invalid token' });
    }

    req.user = user;
    next();
  });
};

// 권한 검증 미들웨어
const authorize = (...roles) => {
  return (req, res, next) => {
    if (!req.user) {
      return res.status(401).json({ error: 'Not authenticated' });
    }

    if (!roles.includes(req.user.role)) {
      return res.status(403).json({
        error: 'Insufficient permissions',
        required: roles,
        current: req.user.role
      });
    }

    next();
  };
};

// 사용 예제
app.get('/api/public', (req, res) => {
  res.json({ message: 'Public content' });
});

app.get('/api/protected', authenticateToken, (req, res) => {
  res.json({ message: 'Protected content', user: req.user });
});

app.delete('/api/admin', authenticateToken, authorize('admin'), (req, res) => {
  res.json({ message: 'Admin action' });
});

app.post('/api/data', authenticateToken, authorize('user', 'admin'), (req, res) => {
  res.json({ message: 'Create data' });
});
```

### 수정안 2: NestJS 가드

```typescript
import { Injectable } from '@nestjs/common';
import { AuthGuard } from '@nestjs/passport';

@Injectable()
export class JwtAuthGuard extends AuthGuard('jwt') {}

@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    const requiredRoles = this.reflector.get<string[]>(
      'roles',
      context.getHandler(),
    );

    if (!requiredRoles) {
      return true;
    }

    const request = context.switchToHttp().getRequest();
    const user = request.user;

    return requiredRoles.some(role => user.roles?.includes(role));
  }
}

// 데코레이터
export const Roles = (...roles: string[]) =>
  SetMetadata('roles', roles);

// 컨트롤러
@Controller('api')
export class DataController {
  @Get('public')
  getPublic() {
    return { message: 'Public' };
  }

  @Get('protected')
  @UseGuards(JwtAuthGuard)
  getProtected(@Request() req) {
    return { message: 'Protected', user: req.user };
  }

  @Delete('admin')
  @UseGuards(JwtAuthGuard, RolesGuard)
  @Roles('admin')
  deleteData() {
    return { message: 'Admin action' };
  }
}
```

### 수정안 3: 클라이언트 에러 처리

```javascript
// fetchWithAuth.js
class AuthError extends Error {
  constructor(status, message) {
    super(message);
    this.status = status;
  }
}

async function fetchWithAuth(url, options = {}) {
  const token = localStorage.getItem('token');

  if (!token) {
    throw new AuthError(401, 'No authentication token');
  }

  const headers = {
    ...options.headers,
    'Authorization': `Bearer ${token}`
  };

  const response = await fetch(url, {
    ...options,
    headers
  });

  if (response.status === 401) {
    // 토큰 만료 - 새로 로그인
    localStorage.removeItem('token');
    window.location.href = '/login';
    throw new AuthError(401, 'Token expired');
  }

  if (response.status === 403) {
    // 권한 부족
    throw new AuthError(403, 'Insufficient permissions');
  }

  if (!response.ok) {
    const error = await response.json();
    throw new Error(error.message || `HTTP ${response.status}`);
  }

  return response.json();
}

// 사용
async function deleteUser(userId) {
  try {
    await fetchWithAuth(`/api/users/${userId}`, {
      method: 'DELETE'
    });
    console.log('User deleted');
  } catch (error) {
    if (error.status === 401) {
      console.log('Please log in again');
    } else if (error.status === 403) {
      console.log('You do not have permission to delete users');
    } else {
      console.error(error.message);
    }
  }
}
```

### 수정안 4: React 컴포넌트 보호

```javascript
// ProtectedRoute.js
import { useAuth } from './AuthContext';
import { Navigate } from 'react-router-dom';

export function ProtectedRoute({ children, requiredRole }) {
  const { user, loading } = useAuth();

  if (loading) {
    return <div>Loading...</div>;
  }

  if (!user) {
    // 로그인 페이지로 리디렉트
    return <Navigate to="/login" />;
  }

  if (requiredRole && !user.roles.includes(requiredRole)) {
    // 권한 부족 페이지로 리디렉트
    return <Navigate to="/forbidden" />;
  }

  return children;
}

// 라우팅
import { BrowserRouter, Routes, Route } from 'react-router-dom';

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/login" element={<Login />} />
        <Route
          path="/dashboard"
          element={
            <ProtectedRoute>
              <Dashboard />
            </ProtectedRoute>
          }
        />
        <Route
          path="/admin"
          element={
            <ProtectedRoute requiredRole="admin">
              <AdminPanel />
            </ProtectedRoute>
          }
        />
        <Route path="/forbidden" element={<Forbidden />} />
      </Routes>
    </BrowserRouter>
  );
}
```

### 수정안 5: 토큰 갱신 로직

```javascript
// authService.js
async function fetchWithTokenRefresh(url, options = {}) {
  let token = localStorage.getItem('token');

  let response = await fetch(url, {
    ...options,
    headers: {
      ...options.headers,
      'Authorization': `Bearer ${token}`
    }
  });

  // 토큰 만료 시 갱신 시도
  if (response.status === 401) {
    const refreshToken = localStorage.getItem('refreshToken');

    if (!refreshToken) {
      // 리프레시 토큰도 없음 - 로그인 필요
      window.location.href = '/login';
      throw new Error('Token expired');
    }

    // 토큰 갱신 요청
    const refreshResponse = await fetch('/api/auth/refresh', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ refreshToken })
    });

    if (!refreshResponse.ok) {
      // 갱신 실패 - 로그인 필요
      window.location.href = '/login';
      throw new Error('Failed to refresh token');
    }

    const { token: newToken } = await refreshResponse.json();
    localStorage.setItem('token', newToken);

    // 새 토큰으로 원래 요청 재시도
    return fetch(url, {
      ...options,
      headers: {
        ...options.headers,
        'Authorization': `Bearer ${newToken}`
      }
    });
  }

  return response;
}
```

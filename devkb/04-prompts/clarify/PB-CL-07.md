# 인증/인가(401 vs 403) 오류 문제 파악

접근이 거부될 때, 그것이 "누구인지 모름(401)"인지 "권한 없음(403)"인지 구분하고 토큰·역할 상태를 기술하도록 유도하는 질문 세트입니다.

## 용도

"접근이 안 돼요"를 넘어, 인증 실패인지 인가 실패인지, 토큰 누락/만료/무효인지 역할 부족인지 좁힙니다.

## 적용 시점

- "401 Unauthorized" 또는 "403 Forbidden"이 발생함
- 로그인은 됐는데 특정 리소스만 막힘
- 토큰 만료 시 처리가 없거나 401/403을 뭉뚱그려 처리함
- 역할(RBAC)에 따라 접근이 갈리는데 동작이 이상함

## 필수 입력

- 응답 상태 코드(401 vs 403)와 오류 응답 본문
- 요청에 붙은 Authorization 헤더/쿠키 상태
- 요청한 리소스와 사용자의 역할(role)

## 프롬프트 템플릿

아래 질문에 답해 인증/인가 중 어느 단계에서 막혔는지 좁혀주세요.

1. **상태 코드 확인**
   - 응답이 401인가요, 403인가요? 오류 본문에 `code`(예: TOKEN_EXPIRED)나 message가 있나요?
   - 같은 요청이 어떤 사용자는 통과하고 어떤 사용자는 막히나요?

2. **인증 정보(401 계열)**
   - 요청에 Authorization 헤더나 쿠키가 실제로 붙어 나가나요? (Network 탭 확인)
   - 토큰이 아예 없나요, 형식이 잘못됐나요, 만료됐나요? 서버가 `TokenExpiredError`를 구분하나요?

3. **권한(403 계열)**
   - 사용자는 인증됐는데(로그인 상태) 특정 작업만 막히나요?
   - 그 리소스/작업에 요구되는 역할은 무엇이고, 현재 사용자의 역할은 무엇인가요?
   - 서버 응답이 required/current 역할을 알려주나요?

4. **처리 흐름**
   - 클라이언트가 401과 403을 다르게 처리하나요? (401→재로그인, 403→권한 안내)
   - 401 시 토큰 갱신을 시도하나요? 갱신 실패 시 어디로 가나요?

5. **재현·범위**
   - 항상 발생하나요, 일정 시간(토큰 수명) 후부터인가요?
   - 특정 엔드포인트만인가요, 보호된 모든 경로인가요?

## 출력 계약

인증/인가 문제 정의:
- 증상: [401 or 403, 오류 code/message]
- 인증 상태: [토큰 없음/형식오류/만료/유효]
- 권한 상태: [요구 역할 vs 현재 역할]
- 클라이언트 처리: [401/403 분기 여부, 토큰 갱신 시도 여부]
- 재현: [항상 / 토큰 수명 경과 후, 대상 경로 범위]

## 셀프 체크리스트

- [ ] 401과 403을 명확히 구분했는가?
- [ ] 요청에 Authorization 헤더/쿠키가 실제로 붙는지 확인했는가?
- [ ] 토큰 상태(없음/만료/무효)를 특정했는가?
- [ ] 요구 역할과 현재 역할을 대조했는가?
- [ ] 토큰 수명 경과 후 발생 여부를 확인했는가?

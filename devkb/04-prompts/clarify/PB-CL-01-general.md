# 일반적인 문제 상황 파악

문제를 명확히 이해하기 위한 기본 질문 프롬프트입니다.

## 용도

초기 문제 분석 단계에서 개발자가 문제를 정확히 설명하도록 유도합니다.

## 적용 시점

- 문제 설명이 불완전함
- "작동하지 않음"만 언급됨
- 에러 메시지 없이 증상만 설명됨

## 필수 입력

- 문제 증상
- 발생 환경 (개발/스테이징/프로덕션)

## 프롬프트 템플릿

당신의 문제를 다음 순서로 설명해주세요:

1. **발생한 현상**: 무엇이 잘못되었는가?
   - 정확한 에러 메시지 (있으면)
   - 예상 동작 vs 실제 동작

2. **발생 환경**:
   - 개발 환경인가, 프로덕션인가?
   - 특정 브라우저/OS에서만인가?
   - 언제부터 발생했는가?

3. **재현 방법**:
   - 어떻게 하면 이 문제가 다시 발생하는가?
   - 특정 조건이 있는가?

4. **기술 스택**:
   - 사용 중인 프레임워크/라이브러리
   - 버전 정보

5. **시도한 해결책**:
   - 이미 시도한 것들은?
   - 그 결과는?

## 출력 계약

명확한 문제 정의:
- 현상: [구체적 설명]
- 환경: [상세 정보]
- 재현: [단계별 방법]
- 기술: [스택 정보]

## 셀프 체크리스트

- [ ] 에러 메시지 전체를 수집했는가?
- [ ] 재현 단계를 명확히 할 수 있는가?
- [ ] 기술 버전 정보를 알고 있는가?
- [ ] 시간대별 패턴이 있는가?

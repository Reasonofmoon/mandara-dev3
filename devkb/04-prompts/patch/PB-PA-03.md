# 제어 컴포넌트 폼 상태 수정 적용

진단된 폼 상태 문제(onChange 누락, defaultValue/value 혼용, 초기화 오류 등)에 대해 제어 입력으로 정규화하는 수정안을 트레이드오프와 함께 제시하고, 데이터 유실 없이 안전하게 구현하도록 지시하는 프롬프트입니다.

## 용도

F-02 플로우의 "6단계: 수정안 선택"을 근거로, 제어/비제어 혼용을 제거하고 입력 상태를 단일 소스로 정규화합니다. 필드 유형(text, checkbox, radio, select)별 수정안과 폼 라이브러리 도입 여부를 비교합니다.

## 적용 시점

- "changing an uncontrolled input to be controlled" 경고 원인이 특정됐을 때
- value는 있으나 onChange가 없는 입력이 식별됐을 때
- 다중 입력 상태를 단일 객체로 정규화할지 결정할 때

## 필수 입력

- 진단된 원인 (예: defaultValue와 value 동시 사용)
- 문제 입력 필드 목록과 각 필드 유형
- 폼 라이브러리 사용 여부 및 초기값 소스

## 프롬프트 템플릿

```
다음 폼 상태 불일치 원인에 대한 수정안을 적용하라.

[진단된 원인]
- 컴포넌트: {폼 컴포넌트명}
- 근본 원인: {예: value만 있고 onChange 누락 / defaultValue+value 혼용}
- 대상 필드: {필드명과 유형: text/checkbox/radio/select}

[요구사항]
1. 아래 수정안 후보를 트레이드오프와 함께 비교하라:
   - 수정안 A: 순수 제어 입력으로 통일 (예측 가능, 리렌더 비용)
   - 수정안 B: checkbox/radio는 checked, select는 value로 정규화
   - 수정안 C: react-hook-form 등 라이브러리 도입 (보일러플레이트 감소, 의존성 추가)
2. 최소 침습 원칙: 초기값과 입력 데이터를 보존한 채, 혼용 패턴만 제거하라.
3. 사이드 이펙트를 점검하라:
   - 초기 렌더에서 undefined→값 전환으로 인한 제어 전환 경고
   - controlled 전환 시 커서 위치/포커스 손실 여부
4. 롤백 가능성 확보: 필드 단위로 변경을 나눠 되돌릴 수 있게 하라.

[출력]
- 선택한 수정안과 근거
- 필드별 수정 전/후 코드 diff
- 사이드 이펙트 점검 결과
- 롤백 절차
```

## 출력 계약

- 선택한 수정안과 근거 (라이브러리 도입 여부 판단 포함)
- 필드별 수정 전/후 코드 diff
- 사이드 이펙트 점검표: 제어 전환 경고, 포커스/커서, 초기값 처리
- 롤백 절차: 필드 단위 되돌리기 방법

## 셀프 체크리스트

- [ ] 모든 제어 입력에 value(또는 checked)와 onChange가 쌍으로 존재하는가?
- [ ] defaultValue와 value를 동시에 사용하는 필드가 남지 않았는가?
- [ ] 초기값이 undefined가 아니어서 제어 전환 경고가 없는가?
- [ ] 입력 중 커서/포커스가 유지되는가?
- [ ] 기존 입력 데이터가 수정 과정에서 유실되지 않는가?
- [ ] 필드 단위로 변경을 롤백할 수 있는가?
